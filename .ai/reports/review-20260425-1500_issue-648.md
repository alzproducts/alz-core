### MEDIUM

- **[`.claude/rules/domain-exceptions.md`] Named-constructor bullet dropped from port** — Status: **Skip (user accepted plan rationale)**. Original `Domain/CLAUDE.md` line 18 said *"Use **named constructors** (`::fromFailedRecords()`) for complex creation logic"* and the issue listed it under "Existing Content to Port". Plan §Tightenings drops it on the basis that zero current Domain exceptions use `from*` named constructors (verified). User confirmed: a rule documenting an absent pattern would drift; canonical pointer would have been fictional.

- **[`.claude/rules/domain-exceptions.md`] `@throws on interface methods` bullet dropped from port** — Status: **Fixed** by creating new rule file `.claude/rules/interfaces.md` (`paths: app/**/*Interface.php`) that ports the original directive into a universally-scoped rule with `MixpanelClientInterface` as canonical example. Drop-and-port chosen over restoring in `domain-exceptions.md` because the rule applies to every interface, not just exception-related ones; firing only when editing an interface is more precise than firing when editing an exception class.

### LOW

- **[`.claude/rules/domain-exceptions.md:12`] `(override the base implementation)` parenthetical was misleading for `\LogicException` extensions** — Status: **Fixed**. Tightened to drop the parenthetical: `\LogicException` extensions like `UnsupportedFieldException` declare `context()` fresh rather than overriding a base method.

- **[`.ai/implementation-logs/648-domain-rules-migration.md`] Filename missing `issue-` prefix** — Status: **Fixed**. Renamed to `issue-648-domain-rules-migration.md` to match the template in `.ai/implementation-logs/CLAUDE.md` and the form referenced in the plan itself.

- **[`app/Domain/CLAUDE.md:20`] Pointer placement reads as appendix to "What NOT to Put"** — Status: **Skip (not raised)**. The blockquote `> Exception design rules → ...` sits between sections without a heading break. Visual nit only. Not surfaced for decision because it's purely cosmetic and matches the established `app/Infrastructure/CLAUDE.md` convention.

## Positive Observations

- Tightenings explicitly documented with verification evidence (regex against actual files) rather than silent edits — meets the issue's "nothing silently dropped" criterion.
- All five canonical class pointers verified to exist with the claimed inheritance chains (`AuthenticationExpiredException → PermanentApiFailure → AbstractApiException → DomainException`) and trait composition (`AggregatesChildResultsTrait` does internally compose `ThrowsOnValidationFailureTrait`, matching the rule's "do not compose both" directive).
- Pointer style in `Domain/CLAUDE.md` matches the convention established by `Infrastructure/CLAUDE.md` post-#647 — consistent precedent across the rule-migration series (#642/#644/#645/#647/#648).
- Glob coverage is well-considered: wide exception glob (`app/Domain/**/Exceptions/**/*.php`) catches all 34 exception files including nested `Api/`, `Data/`, `Infrastructure/`, `Inventory/` and per-context exceptions; assembler glob (`app/Infrastructure/**/Mappers/*ViewAssembler.php`) catches all 5 assemblers without firing on sibling mappers.

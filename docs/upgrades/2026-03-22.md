# Dependency Upgrade Report — 2026-03-22

> PR #339 — Release: dependency upgrades 2026-03-21 (`develop` → `main`)

## Executive Summary

**49 packages upgraded** across PHP and Node ecosystems: 3 major, 20 minor, 25 patch, plus 1 new dependency.

This release brings Google Ads API v23 with AI-powered asset generation and Performance Max channel-level reporting, Laravel 12.55 with `Cache::funnel()` concurrency limiting and `Model::withoutRelation()` for safer queue serialization, and Sentry 4.22 with long-lived runtime context isolation critical for our Octane/Swoole setup.

## Headline Improvements

### 1. Google Ads API v23: Performance Max Channel Reporting & AI Asset Generation
`googleads/google-ads-php` v31.1 → v32.2 gives us access to Google Ads API v20–v23. We can now see _exactly_ where PMax ads serve (Search, YouTube, Gmail, Display, Discover, Maps) and use `AssetGenerationService` for AI-powered text/image asset creation.

### 2. Laravel 12.55: Concurrency Control & Queue Improvements
`laravel/framework` v12.49 → v12.55 introduces `Cache::funnel()` for driver-agnostic concurrency limiting, `Model::withoutRelation()` to prevent circular reference serialization in queued jobs, MySQL `timeout()` for query kill, and PostgreSQL `tsvector` column support.

### 3. Sentry Runtime Context Isolation for Octane
`sentry/sentry` v4.19 → v4.22 adds `RuntimeContext` lifecycle helpers that isolate SDK state per request in long-lived runtimes. This prevents scope data leakage between requests under Swoole — a direct improvement for our production environment.

### 4. PHPUnit Deserialization CVE Patched
`phpunit/phpunit` v12.5.8 → v12.5.12 fixes CVE-2026-24765 (CVSS 7.8), an unsafe deserialization vulnerability in PHPT code coverage handling.

### 5. Predis 3.4: Redis 8.0 Support & 25% Faster Handshake
`predis/predis` v3.3 → v3.4 delivers full Redis 8.0 compatibility, a refactored handshake with up to 25% performance improvement, and built-in automatic retry for transient failures.

## Major Upgrades

### googleads/google-ads-php: v31.1.0 → v32.2.0

The biggest upgrade in this release. Moves us from Google Ads API v18–v22 to v20–v23, unlocking several new capabilities.

- **New features gained:**
  - **Performance Max channel-level reporting** — granular breakdown of where PMax ads serve (Search, YouTube, Gmail, Display, Discover, Maps). Data available from June 2025 onwards.
  - **AI asset generation** — `AssetGenerationService` for AI-powered text and image asset creation.
  - **Audience definition from natural language** — `AudienceInsightsService.GenerateAudienceDefinition` builds audiences from text descriptions.
  - **Minute-precision campaign scheduling** — `Campaign.start_date_time` / `end_date_time` with time component.
  - **Invoice service with campaign-level cost mapping** — granular billing breakdowns.
  - **YouTube video upload** and benchmark services.
  - **Monthly API release cadence** — faster feature delivery from Google going forward.

  - Call Ads (`CallAd`, `CallAdInfo`) removed in API v23.
  - Feed system fully removed — must use Assets instead.
  - `AssetPerformanceLabel` removed from PMax campaigns.
  - API v19 dropped in v32.1.0.
- **New bidding strategies:**
  - `OPTIMIZE_IN_APP_CONVERSIONS_WITHOUT_TARGET_CPA`
  - `OPTIMIZE_TOTAL_VALUE_WITHOUT_TARGET_ROAS`
  - Demand Gen `TargetCPC` bidding.
  - `target_roas_tolerance_percent_millis` for portfolio strategies.
- **Compliance:** EU Political Advertising enforcement on April 1, 2026 (v20–v23). New `contains_eu_political_advertising` field on Customer resource.

## Minor Upgrades

### laravel/framework: v12.49.0 → v12.55.1

Six minor releases with substantial new features:

- **`Cache::funnel()` concurrency limiting** (12.53) — driver-agnostic concurrency control without Redis dependency. Limit concurrent executions of critical sections.
- **`Model::withoutRelation()`** (12.54) — clone model with relations selectively removed without mutating original. Prevents circular reference serialization in queued jobs.
- **MySQL `timeout()` query method** (12.51) — kill hanging SELECT queries: `$query->timeout(5)`.
- **PostgreSQL `tsvector` column type** (12.54) — schema builder support for full-text search columns.
- **`Validator::whenFails()` / `whenPasses()`** (12.51) — fluent validation chaining.
- **`encoding` validation rule** (12.55) — validate file/string encoding.
- **Queue improvements**: `queue:monitor` displays oldest pending job (12.54), composite index on jobs table for faster polling (12.54), SQS FIFO queue retry fixes (12.53), job serialization improvements.
- **Collection helpers**: `hasMany()`, `hasSole()`, `containsManyItems()`, `Arr::partition()`, `Arr::sole()`.
- **`@includeIsolated` Blade directive** (12.51) — include components without parent variable leakage.
- **Bug fixes**: race condition fix on real-time facade cache (12.53), deadlock handling on DatabaseLock release (12.50).

## Security Fixes

| CVE | Severity | Package | Fixed in | Description |
|-----|----------|---------|----------|-------------|
| CVE-2026-24765 | **High** (7.8) | phpunit/phpunit | 12.5.8+ | Unsafe deserialization in PHPT code coverage handling. `cleanupForCoverage()` deserializes `.coverage` files without validation, allowing RCE if malicious files exist before test execution. |
| CVE-2025-64500 | **Medium** (6.9) | symfony/http-foundation | 7.3.7+ | `Request` class improperly interprets PATH_INFO, leading to URLs with paths not starting with `/`. Can bypass access control rules built with `/` prefix assumptions. |
| CVE-2026-24739 | **Info** | symfony/process | 7.4.5+ | Process component does not correctly treat `=` when escaping arguments on Windows MSYS2 environments (Git Bash). |
| CVE-2025-45769 | **Disputed** | firebase/php-jwt | 7.0.0+ | Accepts short HMAC keys without validation. Marked DISPUTED by NVD — key management is the caller's responsibility. |

> **Note:** CVE-2026-23524 (Laravel Reverb RCE) and CVE-2026-26004/CVE-2026-27197 (Sentry server-side) were identified in research but affect different components not included in this upgrade.

## Recommendations

### Action Required
- **Google Ads Feed migration**: Feed system fully removed in API v23. Verify all ad integrations use Assets, not Feeds.
- **Google Ads Call Ads**: `CallAd` / `CallAdInfo` no longer supported. Check if any ad creation code references these types.
- **EU Political Advertising compliance**: Enforcement starts April 1, 2026. Implement `contains_eu_political_advertising` field handling if applicable.

### Worth Exploring
- **`Cache::funnel()`**: Evaluate for rate-limiting queue job processing without Redis-specific primitives.
- **`Model::withoutRelation()`**: Consider adopting in queue job dispatching to prevent circular serialization issues.
- **Sentry `RuntimeContext`**: Verify Sentry config leverages new Octane/Swoole request isolation — may require sentry-laravel config update.
- **Predis automatic retry**: Review Redis connection configuration to take advantage of built-in transient failure retry.
- **Performance Max channel reporting**: New reporting dimension available for Google Ads analytics.

### Packages to Watch
- **Laravel 13**: Many packages in this upgrade already declare `^13.0` compatibility — the ecosystem is preparing for the next major Laravel release.
- **Symfony 8.0**: Several Symfony components now ship v8.0 versions. The `psr-http-message-bridge` already jumped to v8.0.4.
- **PHPArkitect 0.8**: New property hook support may enable additional architecture rules for PHP 8.4 patterns used in this codebase.
- **Psalm JIT disabled by default**: If Psalm analysis seems slower, the `--force-jit` flag re-enables it.

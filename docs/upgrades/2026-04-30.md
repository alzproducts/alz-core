## Executive Summary

Nineteen packages upgraded: 7 patches, 10 minors, and 2 majors. The two majors are `googleads/google-ads-php` (v32 → v33) and `phparkitect/phparkitect` (0.8 → 1.0). The google-ads-php major lifted a long-standing upper-bound constraint on `grpc/grpc`, unlocking a transitive jump from v1.57.0 (Aug 2023) to v1.80.0 (Mar 2026) — roughly 23 minor releases of fixes and security work in one go. The remaining changes are dev-tooling improvements (PHPStan, phpstan-disallowed-calls), Symfony polyfill catch-up, and routine library patches.

The plan in issue #681 originally listed several additional dev packages (`laravel/pint`, `pestphp/pest`, `nunomaduro/collision`, `laravel/horizon`, `barryvdh/laravel-ide-helper`, `tightenco/tlint`, `psalm/plugin-laravel`); those did not land in this PR's lockfile diff and presumably moved to a follow-up.

A small code change accompanied the upgrades: `app/Infrastructure/Mixpanel/MixpanelClientFactory.php` now validates that lookup-table keys are strings and tightens the docblock to `non-empty-array<string, non-empty-string>`. This is consistent with stricter type narrowing required by either `phpstan/phpstan` 2.1.54 or `spaze/phpstan-disallowed-calls` v4.12.0.

## Headline Improvements

1. **Google Ads API uplift via `google-ads-php` v33** — keeps the Ads integration on a current API generation, well clear of upcoming sunset windows. Lifts the historical gRPC version cap.
2. **`grpc/grpc` 1.57 → 1.80 unlocked** — ~2.5 years of HTTP/2 hardening, EventEngine maturity, retry-policy stabilisation, and OpenTelemetry instrumentation. Patches **CVE-2024-7246** (HPACK table poisoning, CVSS 6.3) — the one CVE addressed by this entire PR.
3. **`phparkitect` 1.0 stable** — 0.x to 1.0 is a backward-compatibility commitment, not just a version bump. Adds `HaveTrait`/`NotHaveTrait` rules, recursive `**` glob in `excludePath()`, and improved baseline stability when `allowedNamespaces()` changes. Loosens `webmozart/assert` to `^1.12|^2.0`, removing one of two blockers tracked by issue #621.
4. **`spaze/phpstan-disallowed-calls` v4.12.0** — substantive feature release: new `classPattern` directive for wildcard parameter-class matching, param conditions now apply across `allowExceptIn`/`disallowIn`, better error messaging on malformed `typeString` configs.
5. **CVE-2024-7246 fixed** — proxy-fronted gRPC clients are no longer vulnerable to header-cache poisoning attacks. Mitigated by default in v1.58.3+, well below the 1.80.0 we now run.

## Major Upgrades

### googleads/google-ads-php: v32.3.0 → v33.3.0

- **Why this exists:** support for the latest Google Ads API generation. The PHP client is normally pinned to a specific Ads API version, so a major bump tracks Google's own API schedule rather than expressing a SDK redesign.
- **New features gained:**
  - `CartDataSalesView` resource for cart-data-segmented conversion reporting.
  - Lead-Gen conversion support via GA4 / Firebase integrations.
  - App-campaign extension to `ShoppingProduct`.
  - `AssetSet.travel_feed_data` exposes hotel-centre / merchant-centre / travel metadata.
  - Logical-expression filters on `RETAIL_FILTER_BUNDLE` / `RETAIL_FILTER` criteria (tag-based product filtering).
  - Gender exclusions for Performance Max audience controls.
  - Mutable `VideoResponsiveAdInfo` (video responsive ads now editable post-creation).
  - New `VideoEnhancement` resource for tracking Google-generated vs advertiser-provided video assets.
  - AND/OR custom-audience combinations in trending-insights queries.
- **Breaking changes to be aware of:**
  - Video responsive ads now require `videos`, `logos`, `business_name` fields — any code creating these must supply them.
  - `InsightsAudience.topic_audience_combinations` type moved from `InsightsAudienceAttributeGroup` to `common.InsightsAudienceAttributeGroup`.
  - `LOYALTY_SIGN_UPS` user-list category removed — code referencing it will fail.
  - **The gRPC version cap was removed from `composer.json`** (was `>=1.36.0 <=1.57.0`, now `>=1.57.0`). This unlocked `grpc/grpc` 1.57→1.80 transitive. No runtime impact — `ext-grpc` is not installed, so gax uses REST transport (see Recommendations).
- **Bugs fixed:** non-biddable metric support extended across reporting resources; new conversion-attribution segment for shopping view attribution.
- **New APIs available to us:** any of the resources/fields above can now be requested.

### phparkitect/phparkitect: 0.8.0 → 1.0.0

- **Why now:** the maintainers are stabilising the API contract. This is the "we won't break you in 1.x" milestone after years at 0.x. Released 2026-04-25.
- **New rules / features:**
  - `HaveTrait` / `NotHaveTrait` expressions — could be used in `phparkitect.php` to enforce things like "only Eloquent models may use `SoftDeletes`".
  - `IsNotA` expression (symmetrical complement to existing positive type checks).
  - Recursive `**` glob support in `excludePath()` — better fine-grained scoping.
  - Catch-clause types and `@throws` docblock annotations are now tracked as dependencies, reducing false negatives in layer-boundary checks.
  - PHP core classes (`\Exception`, `\DateTime`, etc.) auto-excluded via reflection — less baseline noise.
- **Constraint changes (and what they unblock):**
  - PHP narrowed from `^7.4|^8` → `^8.0` — aligns with our PHP 8.4 target, no impact.
  - Symfony narrowed from `^3.0|^4.0|^5.0|^6.0|^7.0|^8.0` → `^5.4|^6.0|^7.0|^8.0`.
  - **`webmozart/assert` relaxed from `^1.12` → `^1.12|^2.0`** — this clears one of the two blockers in issue #621. The remaining blocker is `helpscout/api-laravel` still pinning `^1.2`, plus a newly-discovered transitive blocker via `phpdocumentor/reflection 6.6.0` (which `psalm` pulls in and requires `^1.7`).
- **Breaking changes:**
  - PHAR users must now pass `--autoload vendor/autoload.php` (we install via Composer — not affected).
  - In glob patterns, `*` no longer crosses directory separators — use `**`. Audit `phparkitect.php` for any rule paths using `*` to span directories.
  - Root-namespace classes are no longer auto-skipped (this should *help* enforcement by surfacing stray global-namespace classes).
  - Docker image distribution discontinued (we don't use it).
- **DX:** clearer violation diagnostics and more stable baselines through rule evolution.

## Bonus Transitive: grpc/grpc 1.57.0 → 1.80.0

The single largest version jump in this PR is hidden inside a transitive bump. `googleads/google-ads-php` v32 capped grpc at `<=1.57.0`. v33 lifts that cap, and Composer resolves to v1.80.0.

**Note:** Post-merge investigation confirmed `ext-grpc` is not installed — the project uses REST transport via gax auto-detection. The `grpc/grpc` Composer package is resolved but its code is never executed. These improvements would only activate if `ext-grpc` were installed in the future.

- **HTTP/2 hardening (v1.60+):** request validation, settings-acknowledgment enforcement, stream-count protections — closes proxy-chain DoS edge cases.
- **EventEngine maturity (v1.58+):** stabilised POSIX/Windows listener implementation; default for Python by v1.70. Improves async/single-threaded performance consistency.
- **C++17 baseline (v1.70+):** modern language features in the core runtime, ~5% faster channel construction.
- **xDS / proxyless service mesh (v1.60+):** formal protocol support (gRFC A101) — relevant only if we ever move to Istio or Google Cloud Service Mesh.
- **OpenTelemetry metrics (v1.65+):** structured channel/server instrumentation replacing OpenCensus.
- **Pick-first LB default (v1.58):** 10–15% latency reduction on simple topologies.
- **Client idleness (v1.60+):** 30-minute default channel idle timeout — frees connection resources in long-running Octane workers.
- **Service-config retry (v1.57+) stabilised:** `max_attempts`, `initial_backoff`, `backoff_multiplier` configurable per method without code changes.
- **PHP 8.5 fixes (v1.80):** infinite-recursion check that prevents segfaults under PHP 8.5 RC.

## Recommendations

- **~~Verify ext-grpc PECL extension~~ — downgraded to informational.** Post-merge investigation confirmed that `ext-grpc` is not installed in the Dockerfile, CI, or local dev. The `google/gax` library auto-detects this via `extension_loaded('grpc')` and falls back to REST transport (`GapicClientTrait::defaultTransport()`, line 460). The project has been running on REST transport all along — the `grpc/grpc` Composer package is resolved but never executed. No ext-grpc compatibility risk exists from this upgrade. **Performance note:** installing `ext-grpc` would enable gRPC transport automatically (no code changes needed), which is measurably faster for protobuf-heavy APIs like Google Ads — but that's an optimisation opportunity, not a risk.
- **Audit `phparkitect.php` for `*` glob usage that needs to become `**`.** Single-star no longer crosses directory separators in v1.0; rules silently scoped to a single level after the upgrade are a quiet regression risk.
- **Consider adopting `HaveTrait` / `NotHaveTrait` rules** if there are conventions worth enforcing (e.g. only Eloquent models using `SoftDeletes`, or Domain classes forbidden from using framework traits).
- **Update issue #621 (webmozart/assert v2.0):** one of two original blockers is gone (phparkitect now allows `^1.12|^2.0`). Remaining blockers are `helpscout/api-laravel`'s `^1.2` pin and the newly-discovered `phpdocumentor/reflection 6.6.0` transitive via psalm.
- **Confirm any code creating `VideoResponsiveAdInfo`, `InsightsAudience.topic_audience_combinations`, or `LOYALTY_SIGN_UPS` user-list categories** — those are the breaking surfaces from the google-ads-php major. A `make test` pass through CI is the practical verification.
- **The follow-up PR for the deferred dev tools** (pint, pest, collision, horizon, ide-helper, tlint, psalm/plugin-laravel) should be tracked — they were in the original plan but didn't land here.
- **Issues #337, #590, #621** remain the standing deferral set: Laravel 13, PHPUnit 13, webmozart/assert 2.0. Status confirmed in the issue body.
